# Rahul & Tejaswee Wedding Invitation - PRD

## Original Problem
User uploaded wedding-invite HTML and wanted a shareable link. Then requested RSVP form, "Add to Calendar" button, and a renamed link with their names.

## Architecture
- Static HTML invite hosted from `/app/frontend/public/`
- Backend: FastAPI (`/app/backend/server.py`) with MongoDB storing RSVPs
- Frontend: existing invitation HTML + inline JS for RSVP fetch, ICS calendar generation, and Web Share API

A fully custom domain (e.g., `rahul-tejaswee.com`) requires deployment with a custom domain, which is available via Emergent's Deploy feature.

## Features Implemented (Jan 2026)
- Envelope gate with "Aum (ॐ)" button opening full invite (original)
- **Add to Calendar** button — generates `.ics` file (Aug 30, 2026, 9:30 AM IST) for one-click add to Google/Apple/Outlook calendars
- **Share Invitation** button — uses native Web Share API where supported, else copies URL to clipboard
- **RSVP Form** — captures name, attending status (yes/no/maybe), guest count, phone, message; stored in MongoDB `rsvps` collection
- Backend endpoints: `POST /api/rsvp`, `GET /api/rsvp`
